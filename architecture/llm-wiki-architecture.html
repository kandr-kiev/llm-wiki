Local LLM Wiki — Architecture

Three-layer knowledge base: Raw Sources → Synthesized Wiki → Schema & Operations • 418 pages • 704 raw files • 15 tools • 6 cron jobs • Obsidian Vault

Layer 1 — Raw Sources

  • • 704 total files across 5 subdirectories
  • • Immutable external sources (URL, PDF, paste)
  • • SHA256 hash stored on ingest
  • • Subfolders: articles (701), papers (1), transcripts (1), assets (1), configs (0)
  • • Source of truth — never modified after ingest
  • • Filled by 6 cron jobs + manual ingestion

.processed/ — HTML→MD Cache

  • • 16 files — intermediate conversion layer
  • • HTML articles converted to Markdown
  • • NOT version-controlled (in .gitignore)
  • • Regenerated from raw/ on each ingest
  • • Bridge between raw/ and wiki/
  • • Contains github_releases.txt

Layer 2 — Wiki Pages

  • • 418 total pages across 9 subdirectories
  • • Mutable, cross-linked knowledge base
  • • comparisons/ 280★ • concepts/ 60 • entities/ 44
  • • playbooks/ 14 • synthesis/ 7 • templates/ 8
  • • queries/ 2 • references/ 2 • index.md • log.md
  • • comparisons/ in .gitignore (regenerated)
  • • Hermes Agent orchestrates synthesis via integrator.py

Outputs & Automation

  • • 15 Python tools in tools/
  • • integrator.py — main ingestion pipeline
  • • wiki_lint.py — structural health checker
  • • wiki_doctor.py — auto-fix broken wikilinks
  • • 6 cron jobs — every 30m to twice daily
  • • outputs/ — lint-report.md + doctor-report.json
  • • GitHub MCP — repository management

Obsidian Vault

  • • Obsidian Git — version control sync
