Lua. This does not mean your hyprlang config is now broken - it will still work, for a few releases.</p> <p data-svelte-h="svelte-ce5zd0">The entire wiki has been updated to reflect new Lua configs and you are encouraged to migrate over
to Lua.</p> <h3 id="user-defined-layouts" data-svelte-h="svelte-1nzyip5">User-defined layouts</h3> <p data-svelte-h="svelte-1w52txd">With Lua, we’ve also added a Layout API for you to define your own layouts directly in the config. Those
layouts can be set globally, per-workspace, per-monitor, etc. and behave just like regular layouts,
except fully controlled by you.</p> <video src="https://hypr.land/imgs/blog/update55/spiralShowcase.mp4" autoplay muted controls loop></video> <h3 id="various-scrolling-improvements" data-svelte-h="svelte-z6gg2k">Various scrolling improvements</h3> <p data-svelte-h="svelte-1ogd9ia">Scrolling now supports fullscreen windows as part of the tape by default:</p> <video src="https://hypr.land/imgs/blog/update55/scrollShowcase.mp4" autoplay muted controls loop></video> <p data-svelte-h="svelte-16nfy00">As well as a native trackpad gesture via <code>scroll_move</code> in lua.</p> <h3 id="icc-profiles" data-svelte-h="svelte-j1byb3">ICC Profiles</h3> <p data-svelte-h="svelte-15idrqy">You can now load an ICC profile per-output in your config via <code>icc = &quot;..path..&quot;</code> in your config for
color accuracy.</p> <h3 id="renderer-and-cm-improvements" data-svelte-h="svelte-1i8mvln">Renderer and CM improvements</h3> <p data-svelte-h="svelte-mrfh3m">Hyprland will now use FP16 precision and improved CM pipelines by default for Color Managed displays.
This helps with color accuracy, screensharing, and various color-related things.</p> <h3 id="breaking-changes" data-svelte-h="svelte-str3gh">Breaking changes</h3> <p data-svelte-h="svelte-1x6pt8m">A few config option updates:</p> <ul data-svelte-h="svelte-su5zai"><li><code>dwindle:pseudotile</code> has been removed as it wasn’t doing anything</li> <li><code>decoration:shadow:ignore_window</code> has been removed (defaults to enabled)</li> <li><code>render:cm_fs_passthrough</code> has been removed, should be automatic with <code>render:cm_auto_hdr</code></li> <li><code>misc:vfr</code> moved to <code>debug:</code> as it’s a debug variable that should not be changed in prod environments</li></ul> <h2 id="new-stuff-and-fixes" data-svelte-h="svelte-w4maxx">New stuff and fixes</h2> <p data-svelte-h="svelte-rmgyer">As usual, tons of goodies:</p> <ul data-svelte-h="svelte-1ghz8br"><li>A bunch of scrolling goodies: <code>expel</code>, <code>consume</code>, <code>consume_or_expel</code>, wrapping options, and so on.</li> <li>A new <code>auto_consuming</code> flag for binds</li> <li>Device tags</li> <li><code>confine_pointer</code> window rule</li> <li><code>move_into_or_create_group</code> for the move dispatcher</li> <li><code>rotatesplit</code> layoutmsg for dwindle</li> <li>A new live pinch cursor zoom trackpad gesture</li> <li>A new glow decoration for windows</li> <li>And much more…</li></ul> <p data-svelte-h="svelte-1sqtt0m">And the usual fixes, way too many to list…</p> <h2 id="the-raw-release" data-svelte-h="svelte-uzat9m">The raw release</h2> <p data-svelte-h="svelte-1t9l6v2">As always, if you prefer reading more, check the release on <a href="https://github.com/hyprwm/Hyprland/releases/tag/v0.55.0" rel="nofollow">Github</a>.</p> <p data-svelte-h="svelte-12ace87">Cheers,
