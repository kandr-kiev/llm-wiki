                <p>Laravel SMS Catcher is a local development helper by <a href="https://github.com/michal78">Michal Skogemann</a> that captures outgoing SMS notifications. Much like Mailpit does for email, it gives you a visual dashboard to inspect the text body and metadata of your SMS messages while working locally.</p>
<h2><a id="content-main-features" href="#content-main-features" class="not-prose mr-1 text-gray-600 font-normal heading-permalink" aria-hidden="true" title="Permalink">#</a>Main Features</h2>
<ul>
<li><strong>Phone-style preview</strong> — Displays text inside a virtual device mockup with a selector to check how line wrapping looks on different screen widths.</li>
<li><strong>Environment-safe defaults</strong> — Blocks the dashboard on production environments, running only in local or debug modes.</li>
<li><strong>Customizable routing</strong> — Lets you customize the dashboard URL path and wrap routes with your own middleware.</li>
<li><strong>Flat-file storage</strong> — Stores messages in a local JSON file in your logs directory, bypassing the need for database tables or migrations.</li>
<li><strong>Inbox management</strong> — Clears individual messages or wipes the entire inbox from the dashboard.</li>
</ul>
<h2><a id="content-how-it-works" href="#content-how-it-works" class="not-prose mr-1 text-gray-600 font-normal heading-permalink" aria-hidden="true" title="Permalink">#</a>How It Works</h2>
<p>The catcher hooks into Laravel's notification system. When you send a notification via the <code>sms</code> channel, the package intercepts it by calling <code>toSms</code> on the notification class. It then writes the sender, recipient, body, and timestamp to <code>storage/logs/sms-catcher.json</code>.</p>
<p>Writing to a flat JSON file means there are no database tables or migrations to set up. Since Laravel excludes the <code>storage/logs</code> directory from git by default, your test messages won't clutter your repository.</p>
<h2><a id="content-dashboard-and-phone-previews" href="#content-dashboard-and-phone-previews" class="not-prose mr-1 text-gray-600 font-normal heading-permalink" aria-hidden="true" title="Permalink">#</a>Dashboard and Phone Previews</h2>

<h2><a id="content-configuration" href="#content-configuration" class="not-prose mr-1 text-gray-600 font-normal heading-permalink" aria-hidden="true" title="Permalink">#</a>Configuration</h2>
<p>Out of the box, the inbox is available at <code>/sms-catcher</code> whenever your app runs in the <code>local</code> environment or has debug mode turned on. If you need to change these settings, you can publish the configuration file:</p>
<pre><code data-theme="github-light" data-lang="bash" class='torchlight' style='background-color: #fff; --theme-selection-background: #e2e5e9;'><!-- Syntax highlighted by torchlight.dev --><div class='line'><span style="color: #6F42C1;">php</span><span style="color: #24292E;"> </span><span style="color: #032F62;">artisan</span><span style="color: #24292E;"> </span><span style="color: #032F62;">vendor:publish</span><span style="color: #24292E;"> </span><span style="color: #005CC5;">--tag=sms-catcher-config</span></div></code></pre>
<p>The resulting <code>config/sms-catcher.php</code> file lets you:</p>
<ul>
<li>Toggle the catcher using the <code>SMS_CATCHER_ENABLED</code> env variable.</li>
<li>Set a custom URL prefix (<code>route.prefix</code>).</li>
<li>Adjust the middleware stack (<code>route.middleware</code>).</li>
<li>Change where the messages are stored (<code>storage_path</code>).</li>
</ul>
<h2><a id="content-installation" href="#content-installation" class="not-prose mr-1 text-gray-600 font-normal heading-permalink" aria-hidden="true" title="Permalink">#</a>Installation</h2>
<p>You can install the package as a development dependency via Composer:</p>
<pre><code data-theme="github-light" data-lang="bash" class='torchlight' style='background-color: #fff; --theme-selection-background: #e2e5e9;'><!-- Syntax highlighted by torchlight.dev --><div class='line'><span style="color: #6F42C1;">composer</span><span style="color: #24292E;"> </span><span style="color: #032F62;">require</span><span style="color: #24292E;"> </span><span style="color: #005CC5;">--dev</span><span style="color: #24292E;"> </span><span style="color: #032F62;">michal78/laravel-sms-catcher</span></div></code></pre>
<p>To capture your messages, make sure your notification classes have a <code>toSms()</code> method that returns a string, array, or an object containing the message body.</p>
