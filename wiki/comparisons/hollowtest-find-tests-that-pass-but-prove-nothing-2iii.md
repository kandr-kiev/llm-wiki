hollowtest: find tests that pass but prove nothing
[#python](/t/python)
[#testing](/t/testing)
[#opensource](/t/opensource)
[#devtools](/t/devtools)
Coverage can be green while your test suite is empty of proof.
That is not a hypothetical. Stack Overflow’s 2025 developer survey found that a majority of developers spend more time fixing *almost-right* AI output — and tests are one of the places that “almost right” hides best. An AI assistant will happily write a function, then write a test that calls it, prints something, and ends with `assert result is not None`. Coverage goes up. Confidence should not.
I built [**hollowtest**](https://github.com/SybilGambleyyu/hollowtest) to catch those tests statically, before they land in main.
