# abuseofcats.com — a public service announcement about [cat(1)](https://man7.org/linux/man-pages/man1/cat.1.html)
$ cat /etc/app/config | grep -v '^#'
abuse. this cat did nothing wrong.
/\_/\
( ;_; )
> ^ <
$ grep -v '^#' /etc/app/config
humane. same output, one less process.
/\_/\
( ^.^ )
> ^ <
grep can open files by itself. Piping a single file through
cat spawns an entire process whose only job is to copy bytes to a
program that already knew how to read them.
[cat(1)](https://man7.org/linux/man-pages/man1/cat.1.html) is for con**cat**enating files. Let cat be cat.
other cats in danger:
cat file | wc -lwc -l < file
cat file | head -n 5head -n 5 file

cat file | sortsort file
