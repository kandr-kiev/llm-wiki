# Agent Skills Patterns

Патерни використання агентових навичок для покращення якості коду та процесів розробки.

## Концепція

Агентові навички (agent skills) — це спеціалізовані інструкції, які кодують процеси розробки так, щоб AI-агенти дотримувались чіткого шляху під час виконання задач. Ключова ідея: агенти не мають пам'яті про попередні дії, тому потрібні чітко визначені процеси для отримання корисних результатів.

## Основні патерни

### 1. Grill-Me Skill

**Призначення:** Глибоке дослідження ідеї через серію уточнюючих запитань.

**Механізм:**
- Агент задає серію запитань про план розробки
- Кожне питання розкриває нову гілку "дерева дизайну"
- Процес триває доки не буде досягнуто спільного розуміння
- Може тривати 15-50+ запитань залежно від складності

**Використання:**
```
/grill-me
```

**Результат:** Детальне розуміння вимог перед початком реалізації.

### 2. To-PRD Skill

**Призначення:** Перетворення розмови на структурований документ вимог.

**Процес:**
1. Запитати детальний опис у користувача
2. Дослідити репозиторій для перевірки тверджень
3. Інтерв'ювати користувача (за допомогою grill-me)
4. Накидати основні модулі
5. Написати PRD за шаблоном

**Ключовий елемент:** User stories — опис бажаної поведінки системи мовою, з використанням Agile-методології.

### 3. To-Issues Skill

**Призначення:** Розбиття PRD на незалежні задачі для виконання.

**Процес:**
1. Знайти PRD (завантажити якщо потрібно)
2. Дослідити код-бейз
3. **Створити vertical slices** — розбити PRD на задачі, які швидко розкривають невідомі невідомості

**Ключова концепція:** Tracer bullet — кожна задача є тонким вертикальним зрізом, що проходить через всі рівні інтеграції, а не горизонтальним зрізом одного рівня.

### 4. TDD Skill

**Призначення:** Покращення якості коду через Test-Driven Development.

**Процес:**
1. Підтвердити, які зміни інтерфейсу потрібні
2. Спробувати рефакторинг для покращення структури
3. Підтвердити, які поведінки тестувати
4. Спробувати спростити модулі для зручності тестування
5. Написати інтерфейси для тестованості
6. Написати один тест за раз (спочатку тест)
7. Написати код для проходження тесту
8. Подивитись на можливості рефакторингу

**Ключова філософія:** Red-green-refactor створює цикл, який триває доки не буде завершено.

### 5. Improve Codebase Architecture

**Призначення:** Зробити код-бейз дружнім до AI-агентів.

**Процес:**
1. Дослідити код-бейз, шукаючи точки плутанини
2. Знайти місця, де розуміння однієї концепції вимагає стрибків між багатьма файлами
3. Знайти місця, де pure functions були виділені тільки для тестованості
4. Знайти місця, де тісно пов'язані модулі створюють ризики інтеграції
5. Презентувати кандидати для поглиблення модулів

**Ключова ідея:** Якщо у вас поганий код-бейз, AI вироблятиме поганий код у цьому бейзі.
