# Hermes Agent

## Overview

**Hermes Agent** — open-source, self-improving AI agent framework, побудований [Nous Research](nous-research).

**Ліцензія:** MIT
**Дата релізу:** 25 лютого 2026
**Мова:** Python
**GitHub:** 105,000+ stars (станом на липень 2026)

> "Between a Claude Code style CLI and an OpenClaw style messaging platform agent."

## Core Philosophy

Hermes Agent вирішує проблему stateless-агентів через **closed learning loop** — замкнене навчальне коло:

1. Агент виконує складну задачу
2. Оцінює власну продуктивність
3. Витягує repeatable patterns
4. Зберігає їх як structured skills (`SKILL.md`)
5. Наступного разу використовує skill замість zero-shot reasoning

Це створює **"the agent that grows with you"** — система, яка стає кращою з кожною сесією.

## Three Architectural Properties

| Property | Description |
|----------|-------------|
| **Skill Creation** | Успішні задачі абстрагуються в reusable skills — `SKILL.md` файли з procedures, pitfalls, verification steps |
| **Skill Improvement** | Skills оновлюються при нових доказах — кращий підхід автоматично замінює старий |
| **User Modeling** | Persistent representation користувача — preferences, decision history, task patterns — в `USER.md` + SQLite episodic archive |

## Memory System (5 Layers)

| Layer | Type | Persistence |
|-------|------|-------------|
| 1. Short-term inference | Transformer context | Session only |
| 2. Procedural skills | SKILL.md files | Permanent |
| 3. Contextual persistence | Vector store (skills indexing) | Permanent |
| 4. User modeling | Honcho (entity-centric) | Permanent |
| 5. FTS5 search | SQLite full-text search | Permanent |

## Profile System (v0.6.0+)

`hermes -p <profile>` — multiple isolated agent instances on single host.

Кожний profile має окремий `HERMES_HOME` з:
- Config files
- SQLite session databases
- Memory repositories
- Loaded skills
- Messaging gateway PIDs

## Key Differentiators vs OpenClaw

| Aspect | Hermes Agent | OpenClaw |
|--------|--------------|----------|
| **Philosophy** | Agent-first (learning loop) | Gateway-first (routing) |
| **Core abstraction** | Learning loop | Gateway process |
| **Skill portability** | agentskills.io (11+ tools) | Framework-specific |
| **Memory** | Honcho + FTS5 + skills | Graph-based (LanceDB) |
| **Backends** | 6 terminal backends | Single |
| **Language** | Python | TypeScript |
