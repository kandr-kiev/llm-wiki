# Honcho

## Overview

**Honcho** — entity-centric memory library від [Plastic Labs](https://github.com/plastic-labs/honcho).

**GitHub:** https://github.com/plastic-labs/honcho

## Purpose

Represents both users and agents as **"peers"** in a persistent memory system.

## Key Principles

### Peer Model

- Users and agents are modeled as peers
- Asynchronously reasons about peer psychology from messages
- Derives facts and stores them in reserved collections
- **No messages = no reasoning = no memory**

### Evolution Over Time

The model evolves based on interactions:
- Preferences
- Work patterns
- Domain expertise
- Decision history
- Communication style

## Integration with Hermes Agent

Honcho is **Layer 4** of Hermes Agent's 5-layer memory system:

| Layer | Type | Persistence |
|-------|------|-------------|
| 1. Short-term inference | Transformer context | Session only |
| 2. Procedural skills | SKILL.md files | Permanent |
| 3. Contextual persistence | Vector store | Permanent |
| **4. User modeling** | **Honcho** | **Permanent** |
| 5. FTS5 search | SQLite full-text search | Permanent |

## How It Works

1. **Message ingestion** — agent receives messages from user
2. **Async reasoning** — Honcho analyzes messages for user traits
3. **Fact derivation** — extracts preferences, patterns, expertise
4. **Storage** — stores in reserved collections
5. **Retrieval** — agent queries "this user prefers TypeScript"

## Philosophy

> "Hermes models users as entities with derived facts."

**vs CrabTalk's graph memory:**
- CrabTalk: "Last time this user asked about deployment, the agent used this approach"
- Honcho: "This user prefers TypeScript"
